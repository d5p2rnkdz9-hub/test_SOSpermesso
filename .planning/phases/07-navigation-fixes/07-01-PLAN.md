  const history = useTreeStore((s) => s.history);

  <action>
**Rewrite BackButton to always render** with page-context-aware behavior instead of returning `null` when `history.length === 0`.

The BackButton currently lives in StickyHeader (server component) and is the only client component there. It needs to detect which page the user is on and adapt its behavior:

**Page-behavior matrix:**

| Page | Condition | Back Behavior |
|------|-----------|--------------|
| Welcome (`/` or empty pathname) | Always | Navigate browser back via `router.back()` — this handles the case where user came from an external site or another page |
| Tree (first question) | `history.length === 0` AND `sessionStartedAt !== null` | Call `reset()` and navigate to `/` (back to welcome) |
| Tree (mid-question) | `history.length > 0` | Call `goBack()` — TreePlayer handles the slide animation |
| Outcome (`/outcome/*`) | `history.length > 0` | Call `goBack()` then `router.replace('/tree')` |
| Outcome (direct URL, no history) | `history.length === 0` | Navigate to `/` (can't reconstruct session) |

**Implementation steps:**

1. **Add `reset` and `sessionStartedAt` selectors** from useTreeStore:
```typescript
const reset = useTreeStore((s) => s.reset);
const sessionStartedAt = useTreeStore((s) => s.sessionStartedAt);
```

2. **Remove the early `return null`** when `history.length === 0`. The button should always render.

3. **Rewrite `handleBack`** with the following logic:
```typescript
const handleBack = () => {
  const isOutcome = pathname.startsWith('/outcome');
  const isTree = pathname === '/tree';
  const isWelcome = pathname === '/' || pathname === '';

  if (isOutcome) {
    if (history.length > 0) {
      goBack();
      router.replace('/tree');
    } else {
      // Direct URL access — no session to go back to
      router.replace('/');
    }
    return;
  }

  if (isTree) {
    if (history.length > 0) {
      // Mid-tree: go to previous question
      goBack();
    } else {
      // First question: go back to welcome
      reset();
      router.replace('/');
    }
    return;
  }

  // Welcome page or unknown: browser back
  router.back();
};
```

4. **Always render the button** — remove the conditional `if (history.length === 0) return null`:
```tsx
return (
  <button
    type="button"
    onClick={handleBack}
    aria-label={t('back')}
    className="flex items-center gap-1 text-sm font-medium text-foreground"
  >
    <ArrowLeft className="h-5 w-5 rtl:rotate-180" />
    <span className="hidden sm:inline">{t('back')}</span>
  </button>
);
```

**Key constraints:**
- Keep `'use client'` directive
- Keep `rtl:rotate-180` on the arrow for RTL support
- Keep the existing `aria-label={t('back')}` for accessibility
- Keep the `hidden sm:inline` pattern for the text label (icon-only on mobile, icon+text on desktop)
- Use `router.replace` (not `router.push`) for tree/outcome navigation to avoid polluting browser history with intermediate states
- Use `router.back()` only on the welcome page where browser history is appropriate
