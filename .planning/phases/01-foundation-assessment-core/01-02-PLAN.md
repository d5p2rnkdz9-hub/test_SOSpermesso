  <action>
Create question type components following these patterns:

**QuestionCard.tsx** - Wrapper for all question types:
```typescript
interface QuestionCardProps {
  question: Question
  children: React.ReactNode
  questionNumber?: number
}
```
- Clean card layout with question text prominent
- Optional description/helper text below
- Conversational feel (not test-like)
- Uses Card from shadcn/ui

**SingleChoice.tsx** - Radio buttons (SINGLE_CHOICE):
```typescript
interface SingleChoiceProps {
  options: QuestionOption[]
  value: string | null
  onChange: (value: string) => void
  disabled?: boolean
}
```
- Vertical stack of radio options
- Each option as clickable card-style button (not plain radio)
- Selected state with brand-green highlight border
- Uses RadioGroup from shadcn/ui internally

**MultipleChoice.tsx** - Checkboxes (MULTIPLE_CHOICE):
```typescript
interface MultipleChoiceProps {
  options: QuestionOption[]
  value: string[]
  onChange: (value: string[]) => void
  disabled?: boolean
}
```
- Vertical stack of checkboxes
- Card-style clickable areas
- Selected state with brand-green check
- Uses Checkbox from shadcn/ui

**YesNo.tsx** - Binary choice (YES_NO):
```typescript
interface YesNoProps {
  value: boolean | null
  onChange: (value: boolean) => void
  yesLabel?: string  // Default: "Si"
  noLabel?: string   // Default: "No"
  disabled?: boolean
}
```
- Two large buttons side by side
- Italian labels by default
- Clear visual distinction for selected

**TextInput.tsx** - Open text (TEXT):
```typescript
interface TextInputProps {
  value: string
  onChange: (value: string) => void
  placeholder?: string
  maxLength?: number
  disabled?: boolean
}
```
- Textarea with character count
- Italian placeholder text
- Uses Textarea from shadcn/ui

**ProfileSelect.tsx** - Profile cards (PROFILE_SELECT):
```typescript
interface ProfileSelectProps {
  profiles: QuestionOption[]  // Each has label = profile description
  value: string | null
  onChange: (value: string) => void
  disabled?: boolean
}
```
- Large cards with full profile description
- One selectable at a time
- Clear selection indicator
- For Q3: tech proficiency self-assessment

**RankingInput.tsx** - Drag to rank (RANKING):
```typescript
interface RankingInputProps {
  options: QuestionOption[]
  value: string[]  // Ordered array of option IDs
  onChange: (value: string[]) => void
  maxSelections?: number  // e.g., "rank top 3"
  disabled?: boolean
}
```
- Clickable items that move to "selected" list
- Numbered 1, 2, 3 in selected order
- Click to remove from ranking
- For Q6: rank top 3 AI use cases
- NOTE: No drag-drop (complexity) - use click-to-add/remove

**ProgressBar.tsx** - Progress indicator:
```typescript
interface ProgressBarProps {
  current: number
  total: number
  showPercentage?: boolean
}
```
- Horizontal bar with brand-green fill
- Shows percentage only (no "Question 3 of 10" - per CONTEXT.md)
- Smooth animation on progress change
- Uses Progress from shadcn/ui
