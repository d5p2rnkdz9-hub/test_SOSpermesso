Implement `goBackTo` by popping the history stack until the target nodeId is found, then setting currentNodeId to that nodeId. Clear outcomeId when going back. The answers for popped nodes should be preserved (same behavior as goBack). If nodeId is not found in history, do nothing (safety guard).

```typescript
goBackTo: (nodeId: string) => {
  const { history } = get();
  const targetIndex = history.indexOf(nodeId);
  if (targetIndex === -1) return;

  // Pop history to just before the target node
  const newHistory = history.slice(0, targetIndex);

  set({
    currentNodeId: nodeId,
    history: newHistory,
    outcomeId: null,
  });
},
```
